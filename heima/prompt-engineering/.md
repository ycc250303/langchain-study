# 提示词工程

## prompt 技巧

- 详细描述
- 让模型充当某个角色
- 使用分隔符标明输入的不同部分
- 指定任务步骤
- 提供例子
- 使用参考文本（RAG）

## Zero-shot 思想

- 在训练阶段不存在与测试阶段完全相同的类别，但是模型可以使用训练过的知识推广到测试集中的新类别上面
  - 模型已知：马（四脚兽）、熊猫（黑白色）、老虎（有条纹）
  - 告知模型：斑马是四脚兽、有黑白色条纹
- 用于**基于已训练能力**，不提供示例，仅通过语言描述任务的要求、目标和约束，让模型**直接生成结果**
- 如：请判断“这款鸡胸肉饱腹感很强，吃起来不柴，推荐”中用户的情感倾向，输出 正面 或 负面

## Few-shot 思想

- 少样本学习，模型在学习了一定类别的大量数据后，对于新类别，只需少量样本就能快速学习
  - 基于**少量企鹅样本**并结合相似度判断，推论未知图片内含有企鹅
- 主要用于基于少量示例，让模型参考示例回答
